# Polish & Accessibility Pass Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cross-cutting Phase 7 pass over all 13 screens: consolidate every ad-hoc "← Voltar"-style back button into one accessible, correctly-iconed primitive; add automated axe-core coverage across every route; and run the manual checks (reduced motion, contrast, full keyboard operability) that no automated test can substitute for.

**Architecture:** Adds one new shared primitive (`BackButton`) and refactors every page built in the four screen plans to use it instead of a hand-rolled `<button>`. Adds `vitest-axe` as a dev dependency and one consolidated accessibility test that renders all 13 pages and asserts zero axe violations. No changes to routing, stores, or use-cases.

**Tech Stack:** React 19, `lucide-react` (already a dependency), `vitest-axe` (new dev dependency), Vitest + Testing Library.

## Global Constraints

- **Prerequisites:** all five prior plans — Design System Foundation, Onboarding & Consent, Home & Assessment Vertical, Crisis Escalation, Support Surfaces — must be implemented first. This plan touches files from all of them.
- Every interactive element must be keyboard-focusable with a visible focus ring (`focus-visible:ring-2 focus-visible:ring-brand`, the convention established in the Design System plan) — this plan closes the gaps where a page used a raw `<button>` without it.
- Every interactive element must meet a 44×44px minimum hit target.
- The "← Voltar" / "← Início" / "← Sair da demo" pattern across every screen switches from a literal `←` character to the `ChevronLeft` icon, per `design-tokens.md`'s iconography table (← Voltar → `ChevronLeft`/`ArrowLeft`) — this plan was the first point where the inconsistency (literal arrow glyphs from Plans B–E vs. the icon-set rule from Plan A) became visible across the whole app at once, so it is fixed here rather than re-opening five already-merged plans.
- `prefers-reduced-motion: reduce` must disable all animation/transition (already global CSS from the Design System plan — this plan verifies it, does not re-implement it).
- Axe-core scans of isolated component fragments commonly flag a false-positive "region" violation (content not wrapped in a landmark) that doesn't apply to a fragment under test — this plan explicitly disables that one rule when scanning; every other rule stays enabled and a violation must be fixed at the source, never silenced.
- `screens/00-overview.md`'s global conventions require `PrivacyBadge variant="chip"` in the header of every authenticated screen (04-13). Home (04) already has it (Home & Assessment plan). This plan closes the gap for the four screens whose own header row can carry it without fighting their explicit per-screen layout spec: Assessment Select (05), Result (07), Peers (12), Manager (13). It is **deliberately not** added to the one-question-per-screen flow (06 — the progress row is already dense and pacing-critical), the three Crisis screens (08-10 — minimal chrome is intentional for a high-stakes single-decision surface), or Chat (11 — its header already carries the equivalent trust signal via "texto anonimizado antes do envio"). This is a documented exception, not an oversight.

- [ ] **Step 3: Write the implementation**

```tsx
import { ChevronLeft } from "lucide-react";

interface BackButtonProps {
  label?: string;
  onClick: () => void;
}

export function BackButton({ label, onClick }: BackButtonProps) {
  return (
    <button
      type="button"
      onClick={onClick}
      aria-label={label ? undefined : "Voltar"}
      className="flex min-h-[44px] min-w-[44px] items-center gap-1 text-label font-semibold text-muted focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-brand"
    >
      <ChevronLeft size={18} />
      {label}
    </button>
  );
}
```

```tsx
import type { ComponentType } from "react";
import { describe, expect, it, beforeEach } from "vitest";
import { render } from "@testing-library/react";
import { axe } from "vitest-axe";
import { MemoryRouter } from "react-router";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { SplashPage } from "./SplashPage";
import { PrivacyPage } from "./PrivacyPage";
import { ConsentPage } from "./ConsentPage";
import { HomePage } from "./HomePage";
import { AssessmentSelectPage } from "./AssessmentSelectPage";

import { ChatPage } from "./ChatPage";
import { PeersPage } from "./PeersPage";
import { ManagerDashboardPage } from "./ManagerDashboardPage";
import { useConsentStore } from "../../stores/consent.store";

const RESULT_STATE = { scaleType: "PHQ-9" as const, totalScore: 12, max: 27, riskSignal: true };

const SCREENS: { name: string; Component: ComponentType; path: string; state?: unknown }[] = [
  { name: "Splash", Component: SplashPage, path: "/" },
  { name: "Privacy", Component: PrivacyPage, path: "/privacy" },
  { name: "Consent", Component: ConsentPage, path: "/consent" },
  { name: "Home", Component: HomePage, path: "/home" },
  { name: "AssessmentSelect", Component: AssessmentSelectPage, path: "/assessment" },
  { name: "Phq9Assessment", Component: Phq9AssessmentPage, path: "/assessment/phq9" },
  { name: "Gad7Assessment", Component: Gad7AssessmentPage, path: "/assessment/gad7" },
  { name: "AssessmentResult", Component: AssessmentResultPage, path: "/assessment/result", state: RESULT_STATE },
  { name: "CrisisOffer", Component: CrisisOfferPage, path: "/crisis" },
  { name: "CrisisAccept", Component: CrisisAcceptPage, path: "/crisis/connect" },
  { name: "CrisisDecline", Component: CrisisDeclinePage, path: "/crisis/line" },
  { name: "Chat", Component: ChatPage, path: "/chat" },
  { name: "Peers", Component: PeersPage, path: "/peers" },
  { name: "ManagerDashboard", Component: ManagerDashboardPage, path: "/manager" },
];

describe("automated accessibility pass (axe-core)", () => {
  beforeEach(() => {
    useConsentStore.setState({ hasConsented: false, consentedAt: null });
  });

  it.each(SCREENS)("$name has no axe violations", async ({ Component, path, state }) => {
    const queryClient = new QueryClient();
    const { container } = render(
      <QueryClientProvider client={queryClient}>
        <MemoryRouter initialEntries={[{ pathname: path, state }]}>
          <Component />
        </MemoryRouter>
      </QueryClientProvider>,
    );

    // "region" is disabled deliberately: axe flags content-not-in-a-landmark when
    // scanning an isolated fragment (no <main>/<nav> wrapper exists at this scope,
    // that's App.tsx's job) — a fragment-testing false positive, not a real issue.
    // Every other rule stays enabled; a violation there must be fixed at the source.
    const results = await axe(container, { rules: { region: { enabled: false } } });
    expect(results).toHaveNoViolations();
  });
});
```
